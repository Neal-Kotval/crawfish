# Crawfish — Agent Organizations

**The operating system for companies that run on AI agents.**

From solo founders spinning up their first agent team to enterprises managing hundreds of autonomous workers, Crawfish is the platform you build your org on.

---

## The core idea

You shouldn't have to build an agent org from scratch.

Pick a template — startup, dev shop, customer support team, research org — and get a preloaded set of agent containers with sensible defaults. Modify, extend, deploy.

---

## Who it's for

Two motions, same platform.

**Bottoms-up.** Solo founders and small teams who want to run lean. Five agents doing the work of twenty people. Crawfish handles the infrastructure so they just direct the work.

**Enterprise.** Larger orgs that already have agents running in production and need visibility, governance, and cost control across all of them.

---

## Core features

### Organization templates
Prebuilt agent configurations for common org shapes — startup, SaaS, dev agency, support team. Each template is a set of agent containers with defined roles, tools, and communication patterns. Editable, forkable, shareable.

### Dual analytics surfaces
- **Developer side** — tool calls, token burn, latency, error rates, inter-agent communication flow, architecture diagrams.
- **Product side** — task completion, success rates, customer-facing agent performance, output quality.

### Agent Jira board
Work items both humans and agents can read and update. Agents pick up tasks, log progress, escalate blockers. Humans assign, review, close. One board, shared context.

### Organization membership
Invite humans and agents as first-class members. Both have roles, permissions, and a presence in the org. An agent is a coworker, not a tool.

### Hosted file system
Shared context layer — markdown files, docs, knowledge bases — that any agent in the org can read and write. No more re-injecting the same context into every conversation.

### Token optimization built in
Custom MCP tools per org that minimize token burn. Especially relevant under Claude's credit model. The platform actively works to make your agent org cheaper to run as it scales.

### Agent architectures
Choose how your org is structured — flat (peers), hierarchical (managers delegate to workers), pipeline (sequential handoffs), hybrid. Switch architectures without rebuilding from scratch.

### Manager crons
Scheduled agent runs — daily standups, weekly reports, nightly batch jobs, monitoring loops. The manager layer that keeps the org running without human intervention.

### Marketplace
Partner integrations — CodeRabbit for code review, Codex for dev work, Claude Code for engineering workflows. One-click installs that drop a fully configured agent into your org.
