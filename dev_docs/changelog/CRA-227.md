# CRA-227 (AL-DV4) — summonable knowledge objects: `Wiki` (now) / `Rag` (deferred)

## What changed
The Agent Language wants knowledge you *summon* into a loop
(`feature_loop(summon=[arch.readonly(), playbook.frozen()])`). The partial substrate was
`Memory` (a mutable KV handle) and the `Context` artifact (threaded between agents) —
neither a versioned, summonable, narrowable unit. This adds the unit: a new
`crawfish.wiki` module with the `Wiki` knowledge object and the deferred `Rag` retrieval
seam.

### `Wiki` — a versioned, summonable knowledge unit (`wiki.py`)
- **Freezable + content-hashed.** `Wiki(Freezable)` carries typed `pages` (each a
  `WikiPage` reusing `ContextEntry`), an `org_id` tenancy key, and a `Version`.
  `content_sha()` is a **Merkle over page leaves** (`WikiPage.page_sha()`), so a re-hash
  only re-derives changed leaves (the CRA-227 resolved-open item). Two structurally
  identical Wikis collapse to one sha.
- **Copy-on-write edits.** `with_page(title, value, ...)` returns a **new frozen** Wiki
  with a distinct sha and leaves the receiver unchanged; replacing a title overwrites only
  that page. Mutating a frozen Wiki raises `FrozenError`. Pages are **tainted by default**
  and a tainted page stays tainted across a CoW edit.
- **Modes.** `readonly()` summons by pinned sha (returns a `SummonRef`); `mutable()` is
  the train-mode edit handle and is **rejected in eval mode** (a frozen Wiki) — mirroring
  `train()`/`eval()` (CRA-209), where eval mode == frozen.
- **Summonable identity.** `SummonRef` carries the unit id + **pinned content sha**, never
  the body. `export()["checksum"]` is the pinned sha; the page values never appear in the
  export payload (only Merkle leaves: title + sha + trust tier).
- **Consult = data, never instructions.** `consult()` materialises a `Context` whose
  entries are tainted (fluid), so summoned knowledge flows through the fluid-data block and
  can never reach an instruction slot or a static-only Sink (SECURITY.md). Pure
  `(wiki) -> Context`, no model call.
- **Storage via the seams.** `persist()`/`load()` ride the `Store` seam (a `ScrubbingStore`
  redacts secrets on the write — no secret lands unredacted), tenancy-scoped by `org_id`
  (a Wiki in org `a` never loads under org `b`). The product model imports the protocols,
  never a concrete backend.
- **Trust tier (gap S6).** Every page carries a `TrustTier`
  (`TRUSTED`/`COMMUNITY`/`UNTRUSTED`) so a low-trust corpus (e.g. `customer-tickets`) is
  never silently trusted like `repo/src`. The tier only ever **raises** suspicion — it
  never lowers taint.

### `Rag` — the retrieval seam, DEFERRED
`Rag` (retrieval over a content-hashed corpus snapshot) is the larger half and is a
documented follow-on. This ships the **seam only**: a `RagSeam` protocol and a
`RagDeferred` marker. The seam locks in two properties now so the deferred impl can't
regress them: embeddings must route through the secret-scrubbing seam, and retrieved hits
are tainted by default and carry the source page's trust tier (stored-injection-via-
retrieval, gap S6). The index version (corpus-sha + embed-model id + chunker config) is the
content hash, so retrieval over a frozen index is replay-deterministic — but no retrieval,
embedding, or indexing is implemented here (calling it raises `RagDeferred`).

## Why it's safe
- Summoned knowledge is **data, not instructions**: `consult()` entries are tainted, so
  they can never reach an instruction slot or a static-only Sink target.
- A summon enters identity **by pinned sha**; the body is never carried in the summon ref
  or the export checksum, so replay is deterministic and a secret body can't leak through
  the reference surface.
- Persistence routes through the `Store` seam, so a `ScrubbingStore` redacts secrets before
  any write; tenancy is scoped by `org_id` (cross-org isolation is enforced on load).
- `mutable()` is rejected on a frozen (eval-mode) Wiki, preserving the reproducibility
  guarantee — knowledge edits are train-mode copy-on-write only.

## Tests
`packages/crawfish/tests/test_wiki.py` — `with_page` is CoW (new frozen Wiki, distinct sha,
receiver unchanged); identical content collapses to one sha; mutating a frozen Wiki raises;
summoned pages are tainted by default and `consult()` yields a tainted `Context`; trust
tier is carried and never lowers taint; `readonly()` pins by content sha (and seals an
unfrozen Wiki deterministically); `mutable()` is rejected in eval mode but works in train
mode; `export()["checksum"]` is the pinned sha and the body never appears; persist/load
round-trips frozen; org isolation (a Wiki in one org never loads under another); the
`ScrubbingStore` redacts on persist; the `Rag` retrieval seam is present but deferred
(raises `RagDeferred`); `page_sha` is a stable Merkle leaf. Deterministic — no live calls.
