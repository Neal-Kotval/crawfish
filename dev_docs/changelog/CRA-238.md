# CRA-238 — SEC-1: craw code as a trust boundary (provenance + consent re-gate)

**What shipped.** `crawfish/provenance.py` — the *capability* net the eval gate does not
provide. `craw code` is a stochastic agent that authors `Definition`s/prompts; a
prompt-injected generator (fed a poisoned ticket/RAG hit) can author a Definition that
declares an attacker-chosen secret, MCP connection, or consequential Sink target. The eval
gate stops quality *regressions*, not a malicious-but-passing artifact. This adds two
deterministic mechanisms — generation stays stochastic + confined; the gate is pure.

**Provenance.** Every generated artifact records who/what generated it + the source taint,
keyed by the Definition's content sha, persisted as a queryable ledger record + a `METRIC`
emission. `Provenance` (frozen), `ProvenanceLedger` (Store-backed, `org_id`-scoped),
`record_provenance(...)`.

**Consent re-gate (the trust boundary).** `regate_generated(...)` diffs a generated
artifact's STATIC declared capabilities (`declared_capabilities` — secrets/egress by
reference, never a fluid value) against any prior `Grant`. A *newly* declared
secret/MCP/consequential-egress **re-enters the install-time consent gate**
(`secrets.consent_install`) — generated ≠ trusted. The default decider is fail-closed
(`DenyConsent`); a decline raises `ConsentRequired` and records no grant.

**Signing.** `sign` / `is_signed` / `assert_admissible` — an unsigned generated artifact
may not fire a consequential action (`SigningRequired`, fail-closed); a non-generated
artifact (no recorded provenance) is outside the gate.

## Public API (`crawfish.provenance`)
- `Provenance`, `ProvenanceLedger`, `PROVENANCE_RECORD_KIND`, `SIGNATURE_RECORD_KIND`
- `ConsentRequired`, `SigningRequired` (both fail-closed)
- `declared_capabilities(definition) -> Capabilities`
- `record_provenance(definition, *, store, generated_by, source_tainted=False, org_id="local", emit_event=True) -> Provenance`
- `regate_generated(definition, *, store, generated_by, package=None, decider=None, source_tainted=False, org_id="local") -> Provenance`
- `sign(definition, *, store, signer, org_id="local")`, `is_signed(...)`, `assert_admissible(...)`

**Determinism / security.** The artifact hash is the Definition's content sha; the gate
decision is pure given the static declared capabilities + the stored `Grant`. A fluid value
can never appear in the consent surface (the prompt-injection spine). Reuses `secrets.py`
(`consent_install`/`Grant`/`GrantManifest`) — no fork.

**Tests.** `tests/test_provenance.py` (8): provenance recorded+queryable; declared
capabilities are static references; re-gate raises `ConsentRequired` when declined (no grant
recorded); records the grant on explicit consent; no-op when nothing new is declared; an
unsigned generated artifact is inadmissible until signed; a non-generated artifact is
outside the gate; provenance is `org_id`-scoped.
