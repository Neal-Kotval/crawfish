# CRA-230 / R3 — `craw replay --swap`: counterfactual time-travel replay

Re-run a *historical* recorded run against a *candidate* change for near-$0: every
unaffected leaf replays bit-for-bit from its cassette, and only the leaves the change
actually dirtied are re-executed. The visceral demo of the F-1 content-addressed +
cassette substrate.

## What shipped

- `craw replay --cassettes <dir> --swap <from>=<to>` — swap one model (or one decode
  setting carried in the recorded `model` slot). A leaf (one cassette = one model call,
  keyed by the F-1 execution coordinate) is **dirtied** iff its recorded
  `RunResult.model == from`; every other leaf is **clean**.
- **Determinism.** Clean leaves replay byte-for-byte (same recorded `RunResult`, $0, no
  model call). Only dirtied leaves differ; their counterfactual is sourced from an
  `--alt-cassettes` dir (a previously recorded `to` run, deterministic — the test path)
  or, absent that, a deterministic re-stamp placeholder charged `--cost-per-leaf`.
- **Cost-bounded cascade.** The dirtied fraction is reported *before* spending; with
  `--budget` set, a dirtied live cascade whose projected spend exceeds the budget is
  **refused** (no live call), so an upstream-change blast radius stays bounded and
  visible. The command exits non-zero when refused.
- **Tenancy.** `--org` carries `org_id` onto the report (F-1 already folds it into the
  cassette key, so a counterfactual never reads another org's leaves).
- `--json` emits the versioned `craw.replay.v1` schema (original vs. counterfactual per
  leaf, dirtied fraction, spend, over-budget).

## Public symbols

`crawfish.replay_swap`: `SwapSpec`, `LeafDelta`, `SwapReport`, `parse_swap`, `plan_swap`,
`run_swap`. CLI: `craw replay`.
