# CRA-202 вЂ” CL-1: Refine, the verifier-gated iterate-until-goal operator

**What shipped.** `crawfish/refine.py` вЂ” `Refine`, a bounded, metered, durable loop
that runs a producing `Definition` (the `body`), checks each frozen `Output` against
an **external** `StopCondition`, and iterates until satisfied OR a bound is hit. It
makes "keep going until good enough, but never past N tries or $X" the *easy* path,
generalising the three fixed-bound re-run atoms (`EscalatingRuntime`=2Г—,
`Run._repair`=+1, `RetryPolicy`=on-exception) into one goal-driven operator.

**Stop signal is external (CL-1 safety).** Three `StopCondition`s: `RubricThreshold`
(a `Rubric` metric clears `at_least`), `PredicateStop` (a typed predicate), and
`VerifierStop` (delegates to a **gated** `Verifier` from CL-2). An assembly check in
`Refine.__init__` rejects a `VerifierStop` whose critic `Definition` shares the body's
`content_sha()` вЂ” the generator may never critique itself.

**Determinism + metering.** Exactly one stochastic leaf per iteration (the body `Run`,
plus the verifier's own leaf for `VerifierStop`); loop counter, stop check, no-progress
and best-tracking are pure. The **one shared** `CostBudget` is threaded into every inner
`Run` (never a fresh `RunContext`); each call is preflighted against `remaining_usd`, so
the loop stops without exceeding the cap by more than one worst-case call. `spent_usd` is
the true delta charged to the shared budget over the invocation вЂ” **Gap #3 (spent=0.0) is
closed** (read from the budget the concrete runtime charges, not hard-coded 0).

**No-progress (noise-aware).** A `progress()` delta within the calibrated band
(`rubric_std`, F-8) counts as no progress вЂ” compared on the ranking delta, never
byte-identical sha вЂ” and stops after `no_progress_patience`. Bounded by
`max_iters` + budget + cancel + no-progress; **never wall-clock**.

**Security spine.** The prior attempt is fed back under a **static** `feedback_key` as a
**FLUID** input (taint propagates; it stays in the data block, never an instruction slot).
The bound (N, $X), the verifier identity, the `feedback_key`, and the coordinate keys are
static, never derived from fluid input. It mutates nothing: each iteration is a fresh
frozen `Output` via `Output.derive` (CoW); the body stays frozen (eval mode). `produced_by`
on each iteration Output is a **deterministic** coordinate (`body.content_sha()#visit`),
not a volatile `Run.id`, so a second-process resume reproduces a bit-identical Output.

## Public API (new)

- `StopCondition` (ABC) вЂ” `async satisfied(output, ctx, runtime) -> bool`, `progress(output) -> float`
- `RubricThreshold(rubric, *, metric, at_least)`
- `PredicateStop(predicate, *, progress=None)`
- `VerifierStop(verifier: GatedVerifier)`
- `RefineResult` (frozen) вЂ” `output`, `refine_iters`, `spent_usd`, `refine_stopped`, `best_progress`
- `Refine(body, until, *, max_iters, feedback_key="_refine_feedback", no_progress_patience=1,
  rubric_std=0.0, on_stuck="return_best", edge_id="refine", name="refine")`
  - `async execute(seed, ctx, runtime, *, ledger=None, resume=False, produce=None) -> RefineResult`
  - `_checkpoint(...)` durability hook (wired in CRA-204)
- `feature_loop(body, *, until, max_iters, **kwargs) -> Refine` (vision-vocabulary alias)

**Tests.** `tests/test_refine.py` вЂ” first-clearing stop, exhaustion+best, budget meters
real spend + caps (2-call budget в‡’ 2 calls), cooperative cancel, noise-band no-progress,
FLUID feedback + taint propagation, same-cassette identical iters + Output sha, and the
self-critique assembly rejection. All deterministic (scripted charging runtime / replay).

**Deviation (recorded).** The spec's intended `NodeKind="refine"` is not added to the
core `NodeKind` enum (owned elsewhere); `Refine.kind = NodeKind.AGGREGATOR` (reduce-shaped:
many attempts в†’ one chosen Output) with a `node_kind_tag = "refine"` for telemetry/CLI.
