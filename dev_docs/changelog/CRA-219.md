# CRA-219 (OPT-1) — the optimization-plane CLI (+ CRA-222 OPT-4 `craw lock` surface)

## What changed
The audit's sharpest surface gap — "the entire optimization plane is libraries with no
CLI" — is closed. `craw code` drives Crawfish through the shell, so the self-optimizing
app needs the optimization plane reachable from `craw`. Five new subcommands bind the
already-shipped primitives; nothing here re-implements a cost model, a search, or a gate.

### OPT-1 — five optimization subcommands (`cli.py`)
Each registers in `build_parser` and accepts the shared flags
`--budget --seed --org --json --model --live`:

- **`craw eval`** — runs a Benchmark against the (eval-mode, frozen) Definition and gates
  the scores against a named stored baseline (`--baseline`, `--set-baseline`,
  `--tolerance`). Exits non-zero **iff** a regression. `--json` emits per-metric scores,
  per-metric deltas vs the baseline, and the honest OPT-2 cost band
  (`lower_usd`/`expected_usd`/`worst_case_usd`). Drives `gate_against_baseline` +
  `estimate_cost`.
- **`craw tune`** — searches the Definition's knobs (`--models`, `--max-trials`,
  `--cost-per-trial`, `--cost-regularized`) under the cost-regularized `Objective` and the
  variance-aware promotion gate. Runs in **train** mode. Same `--seed` ⇒ byte-identical
  `winner` sha + trial log. `--budget` with a per-trial cost stops with
  `stopped_reason="budget"`. Drives `Tuner.tune`.
- **`craw refine`** — runs the verifier-gated `Refine` loop until a goal/bound. `--until`
  shares one expression DSL over Rubric metrics with the operator
  (`<metric><op><threshold>`, op ∈ {`>=`,`>`}, e.g. `score>=0.95`); `--max-iters` bounds
  the loop. Exits non-zero if the goal was not reached (the bound was a bound).
- **`craw learn`** — one eval-gated `LearningLoop.improve` self-versioning cycle, or
  `--rollback <sha>` to re-activate a prior `VersionRecord` (a pointer move — **no model
  call**). A promotion/rollback emits an audit-trail event (gap B4) reachable by the
  AnomalyEngine circuit breaker.
- **`craw guard`** — mines the `--org` corrections corpus into a `GoldenSet`, distils the
  supplied closed-grammar `--predicate` (never `eval`/`exec`), and synthesizes a
  `HouseGuard` at its *earned* stage (`shadow|warn|block`). A guard cannot self-promote to
  `block`; it earns authority only by clearing the joint precision/coverage gate
  (`--precision-floor`, `--min-coverage`). A blocking synthesis emits an audit event.

**Determinism / safety.** Pure orchestration; all randomness flows through the recorded
`--seed`. `--live` toggles the real `claude -p` backend; the default is the deterministic
MockRuntime (a model-knob → `score` scoring responder shaped to satisfy the Definition's
typed output schema), so there is **no live model call in tests**. `--org` threads
`org_id` to every Store read/write. Optimization commands run in **train** mode; `craw
eval` runs in **eval** mode and asserts a frozen Definition (`guard_consequential`) before
a recorded run. Every `--json` payload carries a versioned `schema`
(`craw.<cmd>.v<JSON_SCHEMA_VERSION>`) and is snapshot-tested.

### OPT-4 — the `craw lock` CLI surface (resolver in `resolve.py`, shipped)
- **`craw lock --dir <project>`** discovers the project's Definition units, builds a
  synthetic root `Candidate` whose `dependencies` pin each discovered Definition at its
  exact version, resolves the transitive closure via `resolve(root, source)`, and writes a
  pinned, committable lockfile (`crawfish.closure.lock`) with
  `write_lockfile`. Every transitive `DefinitionRef` is pinned to an exact version +
  `sha256:` integrity; `closure_sha()` is the small reference a run records.
- **`craw lock --check`** is the CI drift gate: it re-resolves and compares the resolved
  `closure_sha()` against the on-disk lockfile (read data-only, which itself re-verifies
  the recorded `closure_sha` and fails closed on a tampered file). Any drift — or a
  missing/invalid lockfile — exits non-zero.

## Why it's safe
- The `--json` surface never fires a Sink (Sinks are eval-only); these commands drive
  benchmarks/searches, not consequential egress.
- `craw learn`/`craw guard` promotions emit audit-trail events (gap B4).
- Resolution is pure + offline (no network, no model call), deterministic ordering;
  reading a lockfile executes no unit code and fails closed on a closure-sha mismatch.

## Tests
`packages/crawfish/tests/test_cli_optimize.py` — every subcommand parses + dispatches +
honours the shared flags; `craw tune --seed S` is byte-identical across runs; `craw eval`
exits non-zero on a planted regression and emits per-metric deltas; `craw tune --budget`
stops with `stopped_reason="budget"`; `craw learn --rollback` re-activates a prior version
with no model call; `craw guard` distils a predicate into a stage and fails closed on a
malformed one; `craw lock` writes a closure and `--check` detects drift / missing
lockfiles; every `--json` key set is snapshot-tested. Deterministic — no live calls.
