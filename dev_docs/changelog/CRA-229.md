# CRA-229 / R2 — `craw prove --no-injection`: assembly-time non-interference

A pre-flight certificate over a Definition's declared contract: prove that no
`Flow.FLUID` input can reach a consequential static-only Sink target / idempotency slot.

## Which guarantee shipped — read this

This is a **MOONSHOT, spike-first** issue. The timeboxed spike (issue §8.2 — *is the
2-point taint lattice statically decidable over the dataflow graph?*) concluded that a
**sound full-graph non-interference proof is NOT buildable within the box today**: it
needs a first-class, inspectable dataflow graph carrying the summarization/carry
operators, an agent-leaf declassification point, and the bounded-refine/branch fragment —
none of which the Definition exposes as a serialized artifact (wiring is implicit in node
construction).

Per the issue's explicit instruction *"NEVER block the stack on the moonshot,"* this
ships the **ALG-3 conservative static-rejection fallback**, not the sound proof:

> **Guarantee shipped:** `alg3-conservative-static-rejection` — a conservative,
> assembly-time, **fail-closed** check that REJECTS any wiring where a fluid-tainted value
> can reach a static-only Sink target / idempotency key.

It is **sound for the fragment it covers** (it never passes a wiring the runtime
`TargetMustBeStaticError` / `StaticOnlyError` gates would reject) and **incomplete** (it
certifies only the declared fluid→static-slot class, not the absence of *every* injection
path). It **fails closed**: a consequential output mis-declared `Flow.FLUID` — which
could carry model-derived content into a target position — is reported as a suspected
path, never assumed safe.

**The sound proof + the formal ALG-7 conformance suite remain a research-frontier
follow-on (best-effort / deferred).**

## Defense-in-depth (invariant 11)

`prove` is an *additional* assembly-time gate. It **never replaces** the runtime
`StaticOnlyError` / `TargetMustBeStaticError` — those still fire at construction/run time.
The certificate is a pre-flight check so a fluid→static-slot wiring is caught *before* a
run.

## What shipped

- `craw prove --no-injection <def>` — exits **non-zero** on a suspected fluid→static-slot
  path, **zero** when the static check passes.
- `--json` emits the versioned `craw.prove.v1` schema (`guarantee`, `proven`, the
  `fluid_inputs` / `static_slots` surfaces, per-slot `obligations`, and `violations`).

## Public symbols

`crawfish.prove`: `ProofObligation`, `ProofResult`, `GUARANTEE`, `prove_no_injection`.
CLI: `craw prove`.
