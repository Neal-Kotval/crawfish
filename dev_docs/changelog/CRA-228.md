# CRA-228 ‚Äî Git-for-agents: content-addressed, diffable, mergeable Program (R1)

**Milestone 7 ‚Äî Revolutionary capabilities. Bet R1.**

Lifts the content-addressed unit M6 shipped (a frozen `Definition`, hashed by
`Definition.content_sha`, derived copy-on-write through `crawfish.derive`, named + lineaged by
`crawfish.definition_store`) from "two shas are equal / not equal" to a **typed, field-level
diff** and a **three-way merge** ‚Äî so an agent program is reviewable the way code is. This is
the substrate the other revolutionary bets key off (R3 replay, R4 guards-as-program-members,
R5 weights all assume a content-addressed graph one can diff and merge).

## What shipped

New module `crawfish.agentdiff` (pure data ‚Äî no model call, no I/O, no wall clock):

- **`diff(a, b) -> DefinitionDiff`** ‚Äî a typed, field-level structural diff taken over the
  *canonical content payload* (`Definition.content_dict` ‚Äî the fields that fold into the sha).
  Because it diffs what the hash sees, the diff is non-empty **iff** `a.content_sha() !=
  b.content_sha()`. Each differing field is a `FieldChange(path, kind, before, after)` with a
  stable dotted `path` (e.g. `team.agents.reviewer.prompt`, `inputs.ticket.flow`) and a
  `ChangeKind` of `ADDED` / `REMOVED` / `CHANGED`. Keyed lists (agents by `role`, parameters
  by `name`, dependency pins by `id`, policies/mcp by `name`) are re-keyed to identity, so a
  re-order without an edit is not a spurious change. Changes are returned path-sorted
  (deterministic). `DefinitionDiff.is_empty` / `.paths()` for ergonomics.

- **`merge(base, a, b) -> Definition | MergeConflict`** ‚Äî a three-way merge over the lineage
  (`base` is the common ancestor). Per leaf: a one-sided change wins; a same-value change on
  both sides is harmless agreement; a both-sided change to **different** values is a typed
  `FieldConflict` ‚Äî **never silently resolved**. Any conflict makes the whole merge a
  `MergeConflict` carrying the full path-sorted conflict set. A clean merge unflattens the
  merged payload, re-validates it into a `Definition` (type-checked, not hand-assembled), keeps
  `base`'s `id` (stays on base's identity lineage), and re-seals through the same content-hash
  CoW law as composition (`derive.refreeze`) ‚Äî so the result is a **new frozen** artifact with
  a **deterministic** content sha (the same three inputs always merge to the same sha).

## Public symbols

`ChangeKind` (`ADDED`/`REMOVED`/`CHANGED`), `FieldChange`, `DefinitionDiff`, `FieldConflict`,
`MergeConflict`, `diff`, `merge`.

## Security / versioning spine

- Diff/merge are taken over `content_dict`, so a `Parameter`'s `flow` (the static/fluid
  prompt-injection boundary) and a `Policy` (static consequential config) are ordinary
  diffable leaves. A `flow` move is only ever applied when **exactly one** side made it
  (base‚Üíside consent); whenever it collides with the other side's change to the same leaf it
  surfaces as a typed conflict for review ‚Äî a merge can never **silently** widen the boundary.
- Un-versioned mutation stays impossible: `merge` only ever returns a freshly-frozen artifact
  (or a typed conflict), never an in-place edit; `derive.refreeze` stamps the content sha.
- Both verbs carry `org_id` implicitly (a merge is within one tenant's lineage; the inputs are
  recalled within an org by `DefinitionStore` upstream) and never mutate their inputs.

## Tests

`packages/crawfish/tests/test_agentdiff.py` (18 tests): diff of identical Definitions is empty;
diff detects prompt / decode-knob / added-agent / added-input / added-dependency changes; diff
is order-insensitive on agents; diff is a frozen hashable value; clean three-way merge combines
disjoint changes; merge is deterministic; merge keeps base identity; one-sided / agreeing
merges equal that side; policies preserved; divergent field ‚Üí typed `MergeConflict` with the
full path-sorted conflict set; fluid/static boundary moves apply one-sided / surface on
collision.

## Deferred

Merge granularity on prompt **text** (token/line-level three-way over a prompt body, vs the
current whole-field granularity) is deferred jointly with AL-DV3, per the issue's open
question. Lifting the diff/merge from a single `Definition` to a whole multi-Definition
`Program` graph (node+edge structure) builds directly on this leaf-level law and lands with the
graph-level R3 replay demo.
