- Teams: `query { teams { nodes { id key name } } }`
- TeamIssues($teamId,$after): `team(id:$teamId){ issues(first:50, after:$after){ nodes { id identifier title description url updatedAt state{name type} assignee{displayName email} labels{nodes{name}} project{name} cycle{number} } pageInfo{ hasNextPage endCursor } } }`
- normalizeLinearState(type): `type === "completed" || type === "canceled" ? "closed" : "open"`.

From cloud/server/src/lib/jwt.ts (extend — reuse getOrCreateJwtSecret + jsonwebtoken):
- `signOrgToken`/`verifyOrgToken` use audience "dash-sync". Add a SEPARATE audience (e.g. "oauth-state") so an oauth-state token can NEVER authorize the dash surface and vice-versa. Short TTL (e.g. 10m). Payload: `{ sub: userId, orgId, provider: "linear" }`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| linear.app browser redirect → public callback | Untrusted query params (`code`, `state`) cross into a pre-auth route carrying no platform session. |
| API → linear.app | client_secret-bearing token exchange + GraphQL; secret is server-only env. |
| app → DB | Integration rows store live OAuth secrets (access+refresh, plaintext this phase). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-02 | Spoofing / Tampering (OAuth CSRF, forged callback) | GET /api/integrations/linear/callback | mitigate | `state` is a signed JWT (audience "oauth-state", 10m TTL) carrying userId+orgId; callback validates it before any token exchange; tampered/expired state → 400, no Integration written. Distinct audience prevents a dash-sync token from authorizing the callback. Test asserts the 400-no-write path. |
| T-20-07 | Information Disclosure (client_secret / token leak) | exchangeCode/refreshAccessToken, GET /integrations | mitigate | Token exchange + refresh server-side only; client_secret in env, never browser/git; GET /integrations returns only { provider, connected, externalWorkspaceName }; never log code/tokens. Test greps the response for token keys. |
| T-20-08 | Tampering (open redirect) | callback success redirect, redirect_uri | mitigate | redirect_uri is the fixed `LINEAR_REDIRECT_URI` env value; the post-callback success redirect targets a fixed app URL, never a request-supplied value. |
| T-20-09 | Elevation of Privilege (callback bypasses auth) | linearCallbackRouter mounted pre-authMiddleware | mitigate | The route is intentionally public (browser redirect has no auth header) but recovers identity ONLY from the signed state — it performs no action without a valid state, so it is not an open relay. |
| T-20-03 | Elevation of Privilege (cross-org integration/issue access) | integrations + select-team routes | mitigate | requireMember() + queries scoped by resolved orgId; select-team updates only org-scoped projects; non-member → 404. |
| T-20-04 | Information Disclosure | Integration.accessToken/refreshToken plaintext at rest | accept | CONTEXT defers encryption-at-rest. Mitigate: tokens only in dedicated columns, never logged, never in responses. Tracked follow-up. Flag to user at phase close. |
| T-20-SC | Tampering | npm installs | mitigate | No new packages (RESEARCH audit: zero installs). Bare fetch, no @linear/sdk. slopcheck N/A. |
