Datasource is sqlite: `provider = "sqlite"` (comment: "swap to postgresql for prod"). NO native JSON/array types — store JSON as `String`.

Existing Project model (the new fields hang off this):
- `id String @id @default(cuid())`, `orgId String` (FK Org cascade), `githubRepo String?`, `githubRepoId Int?`, `defaultBranch String?`, `isPrivate Boolean @default(false)`, `createdAt DateTime @default(now())`, `updatedAt DateTime @updatedAt`
- compound unique declared as `@@unique([orgId, githubRepoId], name: "org_repo_unique")`

Existing Org model has `projects Project[]`. The new `Integration` relation is added as `integrations Integration[]` on Org; the new `Issue` relation is added as `issues Issue[]` on Project.

Cascade convention (from OrgMember/AgentMeta): `@relation(fields: [orgId], references: [id], onDelete: Cascade)`.

Contract-test harness (cloud/server/tests/contract/setup.ts) force-resets the DB via `npx prisma db push --force-reset --skip-generate`, reading schema.prisma directly. The Wave 0 specs import `app`/`db`/`client`/`asUser` from `./setup.js` and follow the github.spec.ts fetch-stub pattern.
