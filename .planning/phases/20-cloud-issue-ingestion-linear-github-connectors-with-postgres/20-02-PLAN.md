<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → API | Authenticated SPA requests cross into org-scoped routes; `:orgId`/`:pid` are attacker-influenceable path params. |
| API → GitHub | Server-held Clerk OAuth token used to fetch issues; repo "owner/name" is constructed from stored Project.githubRepo, not raw user input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-03 | Elevation of Privilege | GET /:pid/issues, POST /:pid/sync (cross-org/cross-project read = IDOR) | mitigate | `requireMember()` resolves orgId and every query is scoped by resolved `orgId` + `pid` (`findFirst({ where: { id: pid, orgId } })`); non-member collapses 403→404 so org existence does not leak. Test asserts org-A member cannot read org-B project issues. |
| T-20-01 | Information Disclosure | sync/issues responses; GitHub token in logs | mitigate | Token is server-only (Clerk-brokered, never sent to browser); sync response excludes any token field; do not `console.log` the token. Test greps the response for token keys. |
| T-20-06 | Tampering (SSRF) | listRepoIssues URL construction | mitigate | URL built from stored `Project.githubRepo` split into owner/name with `encodeURIComponent` per path segment (existing github.ts pattern); not from raw request body. |
| T-20-SC | Tampering | npm installs | mitigate | No new packages (RESEARCH audit: zero installs). House style bans @octokit/@linear SDKs; bare fetch only. slopcheck N/A. |
</threat_model>
