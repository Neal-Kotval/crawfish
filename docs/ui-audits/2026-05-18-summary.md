## Per-surface verdicts

### Web marketing (`web/`) — 18/35 → fixed P1s, still 18/35 *structurally*

Single page (`Index.tsx`), warm-paper-on-vermillion correct, but **no responsive layout** below 1280px and **two dead IDE marketplace buttons** ("Open VS Code marketplace ↗", "Open Cursor marketplace ↗") that rendered as primary CTAs with no `href`. Hero stats are hardcoded `10,412` / `−35%` / `3.25×` literals. CLI code block had 5 raw hex literals. Fix landed: marketplace buttons disabled with `aria-disabled` + tooltip; hex literals replaced with `--ink` / `--surface` / `--good` / `--ink-mute`; inline `matchMedia + data-layout` toggle collapses both grids below 1100px. P2 backlog: hardcoded hero stats, missing "Sign in" CTA in nav, install card version-string drift.

### Platform SPA (`cloud/platform/`) — 20/35 with strong empties, weak polish

Structurally correct — every page has explicit loading / empty / error states — but every primitive is re-rolled inline (5+ hand-rolled card implementations, 3 primary-button implementations). The onboarding flow had three hex literals (`#e9e4d0` × 2, `#f7f3ea`) for paper-text-on-ink that should be a `--ink-on` token. The `ImportModal` scrim used pure-black `rgba(0,0,0,0.45)` instead of warm-ink. The dev-facade Auth page had a fake email input + "Send magic link" button that silently did the same thing as "Continue with GitHub". The Propose stage of onboarding promised users they could "rename anything … on the next screen" — a screen that didn't exist. The Shell sidebar showed a redundant "No orgs yet" empty state on top of the OrgPicker's full-page empty state. All fixed: hex literals → `var(--paper-2)`; scrim → warm-ink, shadow → `--shadow-lg`; dev-facade trimmed to GitHub-only; rename-promise copy replaced; sidebar empty state hidden when empty. P2 backlog: `<Card>` / `<CfMark>` / `<DevBanner>` component extraction; native `confirm()` replacement; projects-list poller `document.hidden` pause.

### Dash studio (`desktop/dash` — Canvas/Board/Plan/Sessions/AgentCanvas/etc.) — 16/35 → 25–30/35

Most of the cluster is near-production (Board 27/35, Plan 28/35, Sessions 30/35) with no mock leaks. `AgentCanvas.tsx` was the critical outlier — an unreleased hi-fi scaffold whose `seedTasks` array, hardcoded `$3.14 / 42.8k / 13% / 62%` rail KPIs, fabricated agent description ("Routes backend tickets… Hired 14 days ago."), and dead "Attach shell" / "Assign task" buttons would show to every authenticated user on every agent. The `#f7f3ea` hex literal recurred in the AgentCanvas avatar and Settings AccountPanel. Fix landed: every seed value replaced with `<EmptyState>` or `<Eyebrow>No X yet</Eyebrow>`; both buttons disabled with `title="Coming soon"`; hex → `var(--paper)`. Deferred as scope creep: Plan.tsx's 13 legacy `--cf-*` token calls (rolled into the cross-surface Wave-5-prep backlog).

### Dash data views (`desktop/dash` — Analytics/Knowledge/Diagnoses/Files/Crons/etc.) — 22/35

Materially cleaner than Canvas — no seed arrays, no fake agent lists. The user-visible lies were narrower but still P1: `runtimes.tsx` test-result colors used the legacy `--cf-success/danger` aliases (works today through the shim; breaks at Wave 5). `Projects.tsx` had 5 instances of `var(--bad, #b1452f)` hex fallbacks and 2 instances of `var(--line, #e5e1d8)` referencing a non-existent token. `Analytics.tsx` had bare `<div style={{color: "var(--cf-fg-secondary)"}}>loading…</div>` instead of `<EmptyState>`. `HomeDashboard.tsx` rendered `"Hello, !"` for users without a display name. `dashboard.tsx` showed a dead `"—"` "Estimated savings" card regardless of data. All 5 fixed atomically.

### Prior audit (org-workspace, `OrgRoute` + `Canvas`)

Six P1s identified for `Canvas.tsx` were already resolved in commit `590f244`; AgentCanvas pass this cycle closes the dash-side P1s entirely. Platform `OrgRoute.tsx` improvements deferred to next cycle.

## Concurrent-staging incident on `desktop/dash`

`dash-studio` and `dash-data-views` both owned routes inside the same submodule. `cc43ac8` shipped with `dash-data`'s commit message but includes `dash-studio`'s `AgentCanvas.tsx` and `Settings.tsx` edits — those files were staged by `dash-studio`, swept up by `dash-data`'s `git add` + commit. **The code changes are correct; only the commit-message attribution is wrong.** If this team shape is reused, future fan-outs should split the submodule along filesystem-isolated boundaries (e.g. `components/` vs `routes/`) or run isolated worktrees per teammate.

## Golden-flow verdict

> **Can a first-time user complete the path "marketing → platform sign-in → org + project import → dash canvas with real data → dash data views with honest empty states" without hitting a dead end?**

**Mostly yes, with one remaining gap.** The hard blockers from the prior audit (Canvas seed-data leak, fake `+412 tok` chip, hardcoded KPIs) are closed. The new audits closed every remaining P1 mock-data leak (AgentCanvas, dashboard savings card, HomeDashboard greeting), every honesty bug on dead CTAs (IDE marketplace buttons, dev-facade magic-link, AgentCanvas "Attach shell"), every hex-literal drift, and the false rename-promise in onboarding. The platform SPA's empty/loading/error states now render the way the rest of the surface promises they will. The remaining gap is at the marketing→platform handoff: the marketing page still has no "Sign in" CTA in the nav, so a returning visitor must guess that auth lives at a separate subdomain. That's a one-component fix that should ship next cycle, but it is the only end-to-end dead-end left in the flow.
