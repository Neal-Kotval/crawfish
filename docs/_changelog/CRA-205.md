# CRA-205 — C1: Make Router a runnable composition step (branch)

**What shipped.** A `ROUTER` arm in `Workflow._run_step` (`workflow.py`) — replacing the
`TypeError` that made `NodeKind.ROUTER` unrunnable — plus a thin `branch(classifier,
branches, *, name)` constructor and an extended `check_types`. A `Router` is now a
first-class step: each item is classified and dispatched through the **same** `_run_step`
machinery as its chosen branch, so a branch may be a `Sink`/`Batch`/`Filter`/`Aggregator`
and inherits the identical budget/taint/checkpoint guarantees (closes audit Gap #3).

**Mechanism.** The arm iterates items, `route`/`route_async`-es each to `(label, branch)`,
and recurses `self._run_step(branch, [item], ctx, rt)`. Taint and per-item lineage carry
across the branch boundary via `Output.model_copy(update={"lineage": item.lineage})`.
Predicate routing is pure (zero model calls, `spent == 0`); a `Definition`-backed
classifier is one leaf `Run` charged to the shared budget. `raise_if_cancelled` runs per
item.

**Assembly type-check.** `check_types` now type-checks a producer's output against
**every** branch's input (`_check_router`): a branch that cannot accept the input raises
`WireError` at assembly. The branches' shared output schema becomes the next producer
schema (`_router_output_schema`) — non-terminal branches must converge to one compatible
schema or `WireError`; a Router whose every branch is a terminal `Sink` is itself terminal
and contributes no downstream schema. An uncovered classifier label still fails at
construction (`UnroutableLabelError`).

**Security (gap S3).** A classifier label is a **fluid-derived control signal** that gates
*which* static branch fires — it never itself becomes a consequential target or an
idempotency key. This is structurally enforced: the branch set is closed and static at
assembly, so a fluid label can only select among pre-declared (static) targets, never
synthesize a new one. A tainted item routed into a static-only `Sink` keeps its taint
across the boundary (the `Sink`'s own static-target invariant — `TargetMustBeStaticError`
at construction — is unaffected).

## Public API

- `branch(classifier: Classifier, branches: dict[str, Node], *, name: str = "router")
  -> Router` — constructor for a runnable Router composition step.

**Acceptance test.** `test_router_runnable.py::test_router_step_runs_end_to_end_no_typeerror`
(end-to-end Router step, each item dispatched, `spent == 0`); plus `WireError` on an
incompatible branch, `UnroutableLabelError` on an uncovered label, taint-carry, and
divergent-branch-schema rejection.

**Ledger / hash implications.** None — no new persisted state or coordinate scheme.
