# CRA-240 / SEC-3 — ADRs + SECURITY.md update for the spine extensions

Closes Gap B2. CLAUDE.md requires an ADR on any architectural/security fork, but the
language-era spine extensions had no ADR/doc owner and SECURITY.md still enumerated only the
six Phase-1 invariants. This ratifies the new semantics and brings the spine doc current.

## ADRs (`docs/architecture/decisions/`)

- **[ADR 0020](../architecture/decisions/0020-sink-fires-only-in-eval-mode.md) — A
  consequential Sink fires only in eval mode.** A new consequential-action gate (AL-T1): a
  Sink against an unfrozen (train-mode) Definition raises, so every side effect is
  attributable to one reproducible content hash. A summoned Wiki is frozen in eval mode.
- **[ADR 0021](../architecture/decisions/0021-alg3-assembly-time-rejection.md) —
  Fluid→static-sink injection is rejected at assembly time.** Ratifies ALG-3 as a
  first-class, fail-closed build gate and the generator-as-trust-boundary (CRA-238): a
  generated artifact must clear the gate to ship. Honest scope: sound for the covered
  fragment, incomplete, fail-closed.
- **[ADR 0022](../architecture/decisions/0022-aggregate-taint-union-and-declassify.md) —
  Aggregate taint = union; `declassify` is the sole audited upgrade.** No laundering by
  aggregation (a fold is tainted if any input was); the only taint-drop is an explicit,
  audited `declassify` unreachable from a fluid path. Two-point semantics; the full `Grade`
  lattice is deferred behind a spike.

Each ADR records rationale + rejected alternatives and cross-links the tickets that
introduce the semantic (AL-T1/CL-1/C3/TS-1/ALG-6) and the `lang_notes/` decision log.

## SECURITY.md

A new **Agent-language spine extensions (Milestone S)** section documents:

- the new **fluid surfaces** (Refine feedback, Router/Classifier labels, Verifier/Quorum
  verdicts, Wiki/Rag hits, generated artifacts, the correction corpus) and how each is
  bounded — in a table;
- **invariants 7–9** (eval-mode Sink, ALG-3 assembly rejection, aggregate-taint-union +
  declassify), each linked to its ADR;
- an updated **review gate** that pairs the static taint conformance suite (ALG-7) with the
  behavioural operator red-team (CRA-239) and points at the
  [Security-review DoD](../architecture/SECURITY-REVIEW-DOD.md).

## Determinism

Documentation/decision work; no runtime change. The ADRs *ratify* invariants already
enforced by other tickets (AL-T1/CL-1/C3/TS-1/ALG-6/ALG-3) and exercised by the CRA-239
red-team.
