# CRA-206 ‚Äî C2a: Program ‚Äî cyclic-capable surface + cyclic check_types

**What shipped.** `Program` (a `Workflow` subclass) in `workflow.py` ‚Äî a typed directed
graph whose **edges may cycle**. It reuses the `Workflow` kernel (`_run_step`,
`check_types` adjacency, the F-2 ledger); the difference is the *driver*: it walks edges
per item rather than running `for step in steps` once. This is the structural keystone the
durable per-iteration ledger (C2b) and `recurse` (C3) compose onto.

**API.** `Program(name, version)` with `.step(node) -> node` (register a graph node) and
`.edge(source, target, *, when, max_visits, on_stuck, progress, rubric_std,
no_progress_patience)`. A back-edge is `target <= source` (re-enters the cyclic region
`[target .. source]`). The `ap-clerk` shape ‚Äî `extract ‚Üí review` with a `review ‚Üí extract`
back-edge guarded by `when=lambda label, out: label == "mismatch"` ‚Äî is expressible.

**Driver / bounding.** Each item walks its back-edge's region until `when` goes false
(convergence) OR a bound trips: `visit_count >= max_visits`, shared-budget exhaustion
(preflight `remaining_usd <= 0`), cooperative cancel (`raise_if_cancelled` per iteration),
or **calibrated no-progress** (an `Edge.progress` ranking delta within `rubric_std`, F-8 ‚Äî
replacing byte-identical-sha, which is too weak live / too aggressive on replay). On a
bound trip the edge takes `on_stuck` (`dead_letter` | `return_last`). The `BatchExecutor`
`CycleError` is **never** triggered ‚Äî the `Program` driver owns cycles.

**Determinism / versioning.** Each back-edge traversal mints a new content-addressed
version via `Output.derive` (no in-place mutation; `Freezable.__setattr__` rejects edits)
and records its `output_content_sha` (F-0) at the F-2 ledger coordinate. `produced_by` is
a **deterministic** coordinate `{region_version}#{edge_id}#{visit}`, never a volatile
`Run.id`, so replay reproduces the Output sha bit-for-bit. One shared `CostBudget` meters
every iteration (no Gap #3 spend leak). Taint carries across every cycle edge.

**Assembly checks.** `Program.check_types` runs the linear `Workflow` adjacency check,
then per edge: a back-edge **requires** `max_visits` (unbounded ‚áí **`UnboundedCycleError`**
at assembly); the back-edge target must accept the source's output (structural compat via
`crawfish.typesystem`, never string equality ‚Äî `WireError` otherwise); reachability holds
because every step sits on the forward chain.

## Public API

- `Program(*, name="program", runtime=None, version="0.1")` ‚Äî cyclic composition surface.
- `Program.step(node) -> Node`, `Program.edge(source, target, *, when=None,
  max_visits=None, on_stuck="return_last", progress=None, rubric_std=0.0,
  no_progress_patience=1, edge_id=None) -> Edge`.
- `Edge` (dataclass: `source`, `target`, `when`, `max_visits`, `edge_id`, `on_stuck`,
  `progress`, `rubric_std`, `no_progress_patience`, `.is_back_edge`).
- `ProgramResult(output, visits, stopped)`.
- `UnboundedCycleError`.

**Decision (NodeKind).** `Program` is a `Workflow` (a composition, not a leaf `Node`), so
it needs **no new `NodeKind`** ‚Äî it dispatches existing kinds through `_run_step`.

**Decision (loop coordinate / region version).** `loop_id =
compute_loop_id(region_version, item_lineage, edge_id)` where `region_version` folds the
content shas of the region's frozen `Batch` Definitions (falling back to step names for
non-Definition steps). This is a **new derived coordinate scheme** on the existing F-2
`ledger_loop` namespace ‚Äî no schema change, but if `region_version`'s composition ever
changes it shifts every loop_id (a migration, same class as `compute_loop_id`'s own
`_LOOP_ID_VERSION`).

**Decision (cassette distinctness across visits).** Per-visit cassettes stay distinct
**without** an `ExecutionCoordinate`, because each iteration's body input *is* the prior
iteration's derived Output (a feedback loop) ‚Äî a content difference that already salts the
cassette key. The `iter_index` coordinate is only needed when the *same* input is re-run
(best-of-k); a guarded loop does not need it.

**Acceptance test.** `test_program.py` ‚Äî fixed-point convergence, `UnboundedCycleError` /
`WireError` at assembly, budget hard-stop with full per-iteration metering, cancel,
calibrated-no-progress dead-letter, taint-carry, and replay-identical final sha at $0.
