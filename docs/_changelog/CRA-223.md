# CRA-223 — DV-0: extract the content-hash path into `crawfish.derive`

The single load-bearing refactor of Milestone 6. The Tuner's content-hash primitives
(`_content_sha` / `_refreeze` / `_with_agents`) were private to `crawfish.tuner`, but the
public Definition-as-variable API (AL-DV1/2/3), `state_dict` (AL-T2), and the property
algebra all need the **same** content-hash path — two implementations would drift, a
determinism hazard. They are extracted into a new, low-dep `crawfish.derive` module and
re-exported by `crawfish.tuner` (the **same** function objects).

## What moved

- New module `crawfish/derive.py` — low-dep (`pydantic` / `hashlib` / `json` + the
  Definition types; **no** eval / metrics / batch / runtime import, so no import cycle —
  the same discipline as `crawfish/tune.py`).
- `crawfish.tuner` now does `from crawfish.derive import _refreeze, _with_agents`. The
  underscored names are aliases of the public functions, so
  `crawfish.tuner._refreeze is crawfish.derive.refreeze` — there is exactly **one** hashing
  path in the tree.

## Public API added (`crawfish.derive`)

- `content_sha(definition) -> str` — delegates to the canonical `Definition.content_sha()`
  (ADR 0017 / F-5); the single source of hash truth.
- `refreeze(base, mutated) -> Definition` — a frozen copy of `mutated` carrying a fresh
  content-hash `Version.sha`; `base` untouched.
- `with_agents(base, agents) -> Definition` — a deep, **unfrozen** copy with the team agents
  replaced (the caller re-seals via `refreeze`).
- Back-compat aliases `_content_sha` / `_refreeze` / `_with_agents` (same objects).

## Determinism / behaviour

Pure relocation: behaviour-preserving. A regression test pins that `refreeze`'s
`version.sha` is **byte-identical** to `Definition.content_sha()`, and that the Tuner's
private names are the very same function objects (`test_derive.py`). The full suite
(including every existing Tuner test) stays green; the demo `definition.lock` regenerates
unchanged (no demo file touched).
