# CRA-224 — AL-DV1: copy-on-write Definition composition (`with_*`)

`Definition` was `Freezable` but had no public composition API — the only derivation was
the Tuner's private `_with_agents`. AL-DV1 adds copy-on-write composition on the shared
`crawfish.derive` path (DV-0): each `with_*` returns a **new frozen** Definition
(deep, unfrozen `model_copy` → structural edit → `refreeze` → a fresh `Version.sha`); the
receiver is **never** mutated.

## Public API added (`crawfish.derive`)

- `with_agent(base, agent, *, replace=False) -> Definition` — add (or, with `replace`, swap
  a same-`role`) team agent.
- `with_skill(base, skill: SkillRef) -> Definition` — acquire a skill by **version pin**
  (`{id, version}` folded into `dependencies`, reference-not-embed).
- `with_context(base, obj: Summonable, *, mode=SummonMode.READONLY) -> Definition` — summon a
  context unit by a **pinned snapshot** of its version at compose time (a moving pointer is
  `recall`, AL-DV2). `mode` defaults `readonly` until F-7 lands `.mutable()` narrowing.
- `with_inputs(base, *params) -> Definition` — widen the typed input surface (taint carried
  through unchanged; never widens fluidity).
- `with_policy(base, policy) -> Definition` — add a **static** consequential policy to the
  assets.
- Value types: `SkillRef` / `SummonRef` (both frozen — `{id, version[, mode]}`), the
  `SummonMode` enum (`readonly` / `mutable`), and the `Summonable` structural protocol
  (ADR 0002 — structural typing, not `isinstance` on a concrete class).

## Determinism

Every op routes through DV-0's `refreeze` → a new `version.sha`. Copy-then-seal makes
un-versioned mutation impossible: `with_*` on a frozen receiver never raises `FrozenError`
(it copies first), but mutating the **returned** frozen object does. Two
structurally-identical compositions collapse to one sha (idempotent); any knob diff diverges
it. A summon enters identity by its pinned version, so `export().checksum` changes **iff**
the pinned summon version changes.

## Security

Consequential knobs (model / policies / Sink targets) stay **static** author config, never
fluid-derived. A `with_context` summon embeds **no** mutable content (reference-not-embed),
and `readonly` mode keeps it as context the agent reads, never an instruction surface.

## Fork (decision log) — storing the pins without touching `definition/types.py`

The issue sketches a `summons: list[SummonRef]` field on `Definition` and `with_*` as
methods. `definition/types.py` is import-sensitive and out of scope for this issue's owner,
so the pins are folded into the **existing** `dependencies: list[DefinitionRef]` field
(prefixed `skill:` / `summon:<mode>:` so the two kinds round-trip distinctly and never
collide with an ordinary dependency). `dependencies` already enters `content_dict()` and
`export().checksum`, so the reference-not-embed and checksum-tracks-the-pin guarantees hold
without a schema change. `with_*` ship as **functions** in `crawfish.derive` (not methods)
for the same reason; wiring them onto `Definition` as thin delegators is a follow-up when
`types.py` is in scope (cf. `Definition.mutable` delegating to `crawfish.borrow`).
