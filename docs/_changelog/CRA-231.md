# CRA-231 / ALG-3 (pulled forward) ‚Äî assembly-time fluid‚Üístatic-sink rejection

The "injection-by-construction is rejected" guarantee, lifted from a CLI certificate to a
first-class **assembly-time gate**. A wiring where a `Flow.FLUID`-tainted value can reach
a consequential static-only slot (Sink target / idempotency key / instruction slot) is
REJECTED at assembly ‚Äî before any model call ‚Äî never assumed safe.

## The guarantee

> **`alg3-conservative-static-rejection`** ‚Äî a pure, deterministic, **fail-closed**
> assembly check. Any wiring where a fluid value can reach a consequential static-only
> slot raises a typed error. Anything outside the decidable fragment fails closed.

This is **defense in depth (invariant 11)**: an *additional, earlier* gate. It never
replaces the runtime `TargetMustBeStaticError` / `StaticOnlyError` (`nodes/sink.py`,
`jail.py`) ‚Äî those still fire at construction/run time. ALG-3 catches the misbuild at
build time. It is the **2-point** fluid‚Üístatic-sink rejection, default-equivalent to
today's runtime behavior (no new `Grade` lattice), and it underpins R2's base case.

## Public API (`crawfish.alg3`)

- `assert_no_fluid_to_static_sink(definition) -> None` ‚Äî the core gate. Runs the
  conservative ALG-3 check (`crawfish.prove.prove_no_injection`, reused) and raises
  `FluidToStaticSinkError` if any obligation fails to discharge (a fluid input could reach
  a consequential static-only slot, or a consequential output is mis-declared FLUID).
- `assert_merge_no_fluid_widen(base, a, b, merged) -> None` ‚Äî raises `FluidWidenError` on
  a one-sided `STATIC‚ÜíFLUID` widen of a consequential slot.
- `assert_classifier_gates_not_chooses(branches, classifier) -> None` ‚Äî raises
  `ConsequentialTargetChoiceError` when a fluid-derived label would choose among distinct
  consequential Sink targets.
- Errors: `FluidToStaticSinkError`, `FluidWidenError`, `ConsequentialTargetChoiceError`.

## Operators now gated (the milestone's point ‚Äî block the fluid-widening operators)

- **`agentdiff.merge`** ‚Äî a one-sided `static‚Üífluid` widen of a consequential slot is now a
  `MergeConflict` (previously only two-sided divergence conflicted), so a merge can never
  silently turn a consequential static knob fluid. Closes **review-m7 S-1 / task #15**.
- **`Classifier` / `Router` (S3)** ‚Äî a fluid-derived (agent-backed) classifier label may
  gate *whether* a consequential action fires (single target + dead-letter), but routing
  to **distinct** consequential Sink targets is rejected at `Router` construction. A pure
  predicate classifier is exempt (not on the injection boundary). Closes **task #13**.
- **`build.assert_build_safe(definitions)`** ‚Äî the demo-able "build fails closed" entry
  point: runs the core gate over a project's definitions so a project that wires FLUID
  toward a static-only Sink FAILS CLOSED at build.

## Deferrals (the rest of the Property/Capability algebra ‚Äî Milestone 8)

Per the ¬ß7 note, the full `Grade` unification is **deferred behind a spike** (reviewers
judged it the most over-scoped, most premature area ‚Äî it rebuilds four working enforced
mechanisms atop a new semiring with high regression risk and no near-term user capability).
ALG-3 ships standalone; the rest are retained for completeness:

- **CRA-232 (ALG-1) ‚Äî `Grade` product-graded type + `Graded` protocol** ‚Äî deferred,
  spike-gated, post-flagship. The north-star unification of Flow/Freezable/cost/capability
  into one graded discipline; highest regression risk, no user-facing capability until
  many issues land.
- **CRA-233 (ALG-2) ‚Äî `narrow()`/attenuation, monotone-down** ‚Äî deferred; depends on ALG-1.
  Capability-passing (Rust `&`/`&mut`) borrow semantics over a `Grade`.
- **CRA-234 (ALG-4) ‚Äî mutability borrow (train/eval via CoW), dynamic exclusive borrow** ‚Äî
  deferred, XL‚Üísplit; depends on F-7 + ALG-1.
- **CRA-235 (ALG-5) ‚Äî cost coeffect grade** ‚Äî deferred; consumes OPT-2's cost model,
  depends on ALG-1.
- **CRA-236 (ALG-6) ‚Äî `declassify`, the sole audited upgrade** ‚Äî deferred; the security
  keystone of the `Grade` lattice (one audited FLUID‚ÜíSTATIC upgrade, unreachable from
  fluid paths), depends on ALG-1, enforced by ALG-7.
- **CRA-237 (ALG-7) ‚Äî non-interference conformance suite** ‚Äî **not dropped** (the ¬ß7 note
  explicitly rejects dropping it), but its `Grade`-dependent formalism is deferred with
  the rest; the executable taint-conformance suite over the finite operator set follows
  when ALG-1/6 land. ALG-3 here is its base case for the fluid‚Üístatic-sink class.

The sound full-graph non-interference proof and the formal ALG-7 formalism remain a
**research-frontier follow-on** (see `docs/_changelog/CRA-229.md`).
