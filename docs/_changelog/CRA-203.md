# CRA-203 — CL-2: Verifier, the gated external-signal critic

**What shipped.** `crawfish/verifier.py` — a `Verifier` that wraps a critic
`Definition` (own version/knobs, optional `Rubric`) and emits a `verdict(output)`
over a **closed label set with a mandatory `default`** (mirrors
`Router`/`Classifier`; unparseable critic emission ⇒ `default`, never a silent
pass). Gating authority is **typed**: a bare `Verifier` is in `WARN`/`SHADOW` and
**cannot stop a loop**. `Verifier.gated(...)` is the only path that admits a
`GatedVerifier` (a usable Refine stop signal), and only after the critic clears an
**absolute-precision** bar against a decision `GoldenSet` via the F-3
`precision_gate` — which **fails closed**: no baseline (never benchmarked) ⇒
`VerifierNotGated`; below `min_precision` ⇒ `VerifierNotGated`. This closes the
CL-2 safety inversion (the draft's `gate_against_baseline(tolerance=...)` would have
admitted a never-benchmarked critic to *block* production).

**Determinism.** The critic call is a single leaf `Run` (the only stochastic
primitive; replays via cassette under mock/replay). Label parse + precision
computation are pure. Frozen critic + pinned emissions ⇒ identical verdict
sequence (tested).

**Security spine.** The critic emission reaching the verdict body is FLUID and is
parsed *as data* against the static, trusted label set — an injected/undeclared
label cannot widen the set. The label set / `default` / `accept_label` / gating
authority are static, never derived from fluid input. Taint propagates: a verdict
over a tainted (fluid-derived) `Output` is itself tainted.

## Public API (new)

- `Verifier(definition, *, labels, default, accept_label, rubric=None, stage=WARN, name, registry=None)`
  - `async verdict(output, ctx, runtime) -> Verdict`
  - `accepts(verdict) -> bool`
  - `can_block: bool` (always `False` for a bare `Verifier`)
  - classmethod `gated(definition, golden, *, labels, default, accept_label, min_precision, decide=None, store=None, baseline_name=None, rubric=None, name, registry=None) -> GatedVerifier`
- `GatedVerifier(Verifier)` — stage `BLOCK`, `can_block == True`, carries `measured_precision`.
- `Verdict` (frozen dataclass: `label`, `tainted`, `source_output_id`, `lineage`).
- `VerifierStage` enum (`SHADOW`/`WARN`/`BLOCK`).

Re-exported from `crawfish` top level (`Verifier`, `GatedVerifier`, `Verdict`,
`VerifierStage`).

## Hash / migration

No content-hash bump. Reuses the existing F-3 gate algebra (`precision_gate`,
`VerifierNotGated`, `load_baseline`) and the `Output` taint/lineage fields
unchanged. No store schema change.

## Dependencies

- F-3 (`precision_gate`, `VerifierNotGated`) — the absolute-precision gate.
- F-4 decision `GoldenSet` — `gated()` accepts an authored decision set (each case
  carries the critic's label as `output` and the ground-truth label as `label`).
- Consumed by CL-1 (`Refine`) as the `VerifierStop` source.
