# CRA-220 — OPT-2: cost model that sees escalation/repair/retry/refine (honest interval)

`estimate_cost` was a silent lower bound — one run per agent per item, blind to the
re-run multipliers of escalation (the 2× strong-model tail), repair (+1), retry (≤n)
and `Refine` (`max_iters`). The advertised cost was therefore a falsely-precise point
that could never overshoot. CRA-220 makes the preview an **honest band**.

## What changed (additive — `total_usd` semantics frozen)

`cost.py` is the single owner of the cost model (F-6); CL-3 / ALG-5 consume it.

- `CostEstimate` already carries the three-number interval — `total_usd` (lower bound,
  unchanged), `worst_case_usd`, `expected_usd` with a `expected_lo_usd` / `expected_hi_usd`
  CI band. Invariant enforced: `total <= expected_lo <= expected <= expected_hi <= worst`.
- `CostShape` + `compose_cost` fold a nesting of operators multiplicatively along the
  operator nesting: `worst_case = lower × Π(per-operator factor)`, escalation re-priced on
  the strong model (`1 + strong_price/base_price`). `expected` uses measured rates as a
  CI-aware band; with no rates `expected == worst_case` (never undercount).
- **New `CostShape.from_runtime(runtime, *, model_prices, config)`** — the OPT-2
  "Open (resolved)" helper. Walks the assembled `_inner` wrapper chain and emits one shape
  per cost-bearing runtime wrapper (`EscalatingRuntime`, `QuorumRuntime`) **outermost-first**,
  exactly the order `compose_cost` folds. So a `Quorum(5)` over an `Escalating(2×)` over a
  leaf previews `5 × 2 = 10×`; escalation is re-priced from `model_prices` (degrading to the
  count-based 2× when no prices). `Refine` (a Node), `Retry` (a policy) and `recurse` (a
  control shape) are not runtime wrappers — the caller folds those alongside the inferred list.
- **New `CostShape.repair()`** — the `Run._repair` `+1` re-prompt as a 2× worst case, with an
  optional measured failure rate for the expected band.

## Determinism

Pure static analysis: `from_runtime` traverses the wrapper chain and reads declared bounds;
no `run()`, no model call, no ledger sampling. Same assembled runtime + base ⇒ identical
three numbers.

## Acceptance

`packages/crawfish/tests/test_cost_interval.py` — escalation priced on the strong model and
≥ lower bound; the nested `Refine∘Escalate∘Quorum` 40× product through the inferred path;
`escalation_rate=0.2` ⇒ expected strictly inside `(lower, worst)` with a CI; no rates ⇒
`expected == worst_case`; `from_runtime` deterministic with no model call. The bare
composition law remains pinned by `test_cost_composition.py`.
