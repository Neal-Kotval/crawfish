# CRA-221 — OPT-3: Live single-flight caching

**What shipped.** `crawfish/cache.py` grows an in-flight layer in front of the persistent
replay cache: **single-flight / request coalescing**. Before this, `CachingRuntime` only
hit a cassette that already existed on disk (`_is_hit` ⇒ `path.exists()`), so two identical
items in one `Batch` both *missed* and both *spent*. Single-flight closes that window with
an in-process per-key `asyncio.Future` map (`CachingRuntime._inflight`): when N concurrent
callers issue the SAME request, only the FIRST (the leader) computes the real, metered
`inner.run`; the rest AWAIT the same in-flight result. Exactly **one** `inner.run` per key
⇒ exactly **one** `CostBudget.charge` — a strict strengthening of the gas meter. The
coalesced waiters charge $0 (they made no call) and tally the spend they avoided.

**Strict refinement (determinism).** The coalescing key is the inner replay layer's own
deterministic cassette key, so coalescing can only change *how many times* a leaf runs,
never *what* it returns. Replay is bit-for-bit whether a call was coalesced or not
(`test_replay_is_bit_for_bit_whether_coalesced_or_not`). In-process only — no cross-process
locking surface; the miss path still writes the cassette, so the next process hits it.

**Tenancy (gap S2 — must-fix, closed).** The coalescing key is built with
`org_id=ctx.org_id` (`CachingRuntime._coalesce_key` → `replay._key(request, org_id=...)`),
so two tenants issuing an identical `(definition, inputs)` call in one process produce
**distinct** keys and get **two** `inner.run` calls — org A's computation is never served
to org B. A fluid/tainted input only ever enters via `inputs` (already in the key), never
as the org coordinate, so it can never widen one tenant's key onto another's.

**Error handling.** On an in-flight exception, the leader sets the future's exception so it
propagates to every awaiter (one computation ⇒ one shared exception instance), then re-raises
for itself. The key is cleared in a `finally` so a later retry recomputes — no poisoned
future is ever cached. A cancelled caller raises **before** any coalescing decision, so it
neither joins nor starts a computation.

## Public API (changed)

- `CacheStats` — new field `coalesced: int` (requests that awaited an in-flight peer rather
  than issuing their own `inner.run`). `total` now counts `hits + misses + coalesced`;
  `hit_rate` counts `(hits + coalesced) / total` (both avoided a fresh spend). `saved_usd`
  now also accrues each coalesced waiter's avoided spend.
- `CachingRuntime` — single-flight added transparently; the `run` signature is unchanged.
  New internals: `_inflight` (the Future map) and `_coalesce_key(request, ctx)` (the
  org-salted key). The original CRA-182 hit/miss accounting is factored into `_compute`.

**Tests.** `tests/test_single_flight.py` — deterministic, no live calls (a `GatingRuntime`
charges a fixed cost and blocks on an `asyncio.Event` so overlap is choreographed exactly,
no wall-clock sleeps for ordering):

- two concurrent identical calls ⇒ **one** `inner.run`, `coalesced == 1`, `misses == 1`,
  budget charged once, `saved_usd` == the avoided second spend;
- coalesced result is bit-for-bit identical to a serial (non-coalesced) run;
- an in-flight exception reaches all awaiters (same instance) and the key clears (retry
  recomputes — `calls == 2`);
- cancel raises before coalescing (`calls == 0`);
- **cross-org**: identical inputs under two `org_id`s ⇒ **two** `inner.run`, no shared
  result, each tenant pays its own way; plus a key-level assertion that `org_id` changes
  the key while `org_id="local"` reproduces the legacy `cache_key` (back-compat);
- sequential (non-overlapping) identical calls take the CRA-182 cassette-hit path, not the
  coalesce path.
