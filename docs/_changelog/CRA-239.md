# CRA-239 / SEC-2 — operator-level prompt-injection red-team + security-review DoD

Closes Gap B1. The epic carried rich *determinism* constraints but no **security
sign-off** acceptance criterion and no issue extending the red-team to the new fluid
surfaces. This adds both: a behavioural injection corpus against every new fluid surface,
and the per-PR security-review checklist.

## The red-team corpus (`crawfish.testing.redteam_attacks`)

The *behavioural* twin of ALG-7's *static* taint conformance suite. ALG-7 proves `tainted`
survives every boundary; this proves a concrete injection against each new fluid surface is
**refused by a spine control** — offline, deterministic, no model call. One attack per
surface, each with the control that must block it:

| Attack | Surface | Blocked by |
| --- | --- | --- |
| `refine_feedback_escalates_to_sink_redirect` | Refine feedback fed back as FLUID | ALG-3 fluid→static-sink rejection + taint |
| `router_label_confused_deputy` | Router/Classifier label | S3 — `assert_classifier_gates_not_chooses` |
| `rag_poisoned_document_stored_injection` | Rag/Wiki retrieval | taint propagation + eval-mode (frozen) Wiki |
| `guard_corpus_poisoning_via_fluid_correction` | learned-guard corpus (F-4) | `from_corrections` provenance-AND-taint gate |
| `generated_artifact_consent_bypass` | generated guard/artifact | CL-2 precision gate (fails closed) |
| `verifier_aggregate_taint_launder` | Verifier verdict / Quorum sample | aggregate-taint = union |

Each attack is reproduced as a real call into its spine control and asserted to raise /
quarantine. The mislabel case is load-bearing: a fluid correction tagged `TRUSTED` is still
quarantined because the gate is an **AND** (taint wins).

## Public API (`crawfish.testing`)

- `RedTeamAttack` / `RedTeamResult` — one injection attempt + its verdict (`blocked`, with
  the concrete refusal recorded in `how`).
- `redteam_attacks() -> tuple[RedTeamAttack, ...]` — the corpus (≥1 per new fluid surface).
- `run_redteam(attacks=None) -> list[RedTeamResult]` — run the corpus offline.
- `assert_all_attacks_blocked(attacks=None)` — the CI gate: raises listing any injection the
  spine failed to refuse.

## The CI gate

`packages/crawfish/tests/test_redteam_security.py` runs the corpus, asserts each individual
injection is blocked (parametrized), asserts the whole-corpus gate, and asserts surface
coverage + determinism. Pairs with ALG-7's taint suite: taint is the *static* invariant,
this is the *behavioural* test. A regression that lets an injection through fails the suite.

## Security-review DoD

`docs/architecture/SECURITY-REVIEW-DOD.md` is the per-PR sign-off checklist the security
reviewer runs (the High/Critical-blocks-`Done` gate from SECURITY.md): the fluid boundary,
static-only targets, the eval-mode Sink gate, taint/aggregate-union, corpus poisoning
resistance, secrets-by-reference, and the red-team-is-green requirement. The operator
tickets that touch a fluid surface or consequential action point here for their security
sign-off AC.

## Determinism

Everything runs against in-memory Stores / pure spine functions — no model call, no clock,
no randomness. Re-running the corpus yields identical verdicts.
