# docs/

Long-form documentation for Crawfish. Five sub-folders, each scoped to a single audience.

## Layout

- **`product/`** — what Crawfish *is*. Vision, design system, feature inventory, integration matrix.
  - `DESIGN.md` — design tokens, color, typography, component primitives.
  - `BRAINSTORM.md` — ranked half-formed ideas mapped to the platform.
  - `FEATURES.md` — running inventory of what's shipped per package.
  - `INTEGRATIONS.md` — runtime adapter priority + integration shape.

- **`roadmap/`** — what we're building and when.
  - `GRAND_PLAN.md` — 18–24 month vision (Stage 1 → Stage 2).
  - `ROADMAP-MVP.md` — MVP-cut slice of the full roadmap.
  - `ROADMAP-WAVE3.md` — Wave 3 trust pass (e2e, contracts, code review).

- **`ops/`** — how to run the system and the team.
  - `RUNNING.md` — how to start every component locally.
  - `AGENT-TEAMS.md` — multi-teammate working conventions.
  - `BENCH-PROTOCOL.md` — token-burn benchmark protocol.
  - `SKILL_SETUP.md` — Claude Code skill / plugin manual steps.

- **`reviews/`** — point-in-time code reviews.
  - `REVIEW-WAVE2.md` — Wave 2 server scaffold + platform review.

- **`superpowers/`** — superpowers-skill plans + specs. Append-only.

- **`specs/`** — contract specs (org, preflight, p4 architecture).
- **`teams/`** — phase team notes + spawn prompts.
- **`screenshots/`** — referenced by `../README.md`.

The active build schedule lives at `../ROADMAP.md`. The repo map lives at `../PRODUCT.md`.
